## [2.9.3-AI] — 2026-03-17

### Додано
- **Плагін Language Translator** (`plugins/lang-translator/`) — перекладач сайту на льоту без API-ключа:
  - Підтримка 10 мов: EN, PL, DE, FR, ES, IT, CS, SK, RO, HU
  - Три позиції switcher: Navbar (пункт меню `language_switcher`), Footer, Floating (фіксована кнопка)
  - Три стилі кнопки: прапор + код / тільки прапор / текстовий dropdown
  - Кеш перекладів у `localStorage` (1–168 год, налаштовується)
  - Паралельні HTTP-запити до Google Translate (до 3 одночасно) — вдвічі швидше за послідовні
  - Прогрес-бар вгорі сторінки під час перекладу
  - Автоінвалідація кешу при зміні налаштувань (`config_hash`)
  - Налаштування через Адмінка → 🧩 Плагіни → ⚙ Language Translator
- **Кнопка «↑» прокрутки вгору** (`views/layouts/base.twig`) — з'являється після 300px прокрутки, розміщена праворуч; якщо активний Floating перекладач — автоматично зміщується під нього

### Змінено
- `assets/js/lang-widget.js` — v5.1: dropdown рендериться в `<body>` з `position:fixed` і динамічним розрахунком позиції (відкривається вгору якщо мало місця знизу); підтримка Footer і Floating позицій
- `templates/translate.php` — розширено список дозволених мов з 3 до 10
- `index.php`, `templates/page.php` — підключено `data/plugins.php`; `LT_CONFIG` передається у Twig-контекст через `lt_config_json`
- `data/ViewContext.php` — додано `applyPluginFilters()` для розширення Twig-контексту плагінами
- `admin/support_reply.php` — розробник тепер може закрити тікет з форми відповіді кнопкою «🔒 Закрити тікет»; SMTP читається з `.env` замість `email_config.php`

## [2.9.2-AI] — 2026-03-16

### Виправлено
- `templates/login.php` — після успішного входу тепер записується рядок у таблицю `user_sessions`; без цього Dashboard завжди показував «0 онлайн» і порожні «Останні входи»

- `data/migrations.php` — версія 10: додано `CREATE TABLE IF NOT EXISTS user_sessions` з індексами; гарантує створення таблиці на існуючих БД після оновлення

## [2.8.0-AI] — 2026-03-16

### Додано
- **Система плагінів** (`data/plugins.php`) — хук-архітектура WordPress-стилю:
  - Actions: `fly_add_action()` / `fly_do_action()`
  - Filters: `fly_add_filter()` / `fly_apply_filters()`
  - Менеджер плагінів в адмінці (`admin/plugins.php`) — вмикання/вимикання, встановлення через ZIP
  - Хуки: `cms.init`, `cms.post.saved`, `cms.page.saved`, `cms.user.login`, `cms.post.content`, `cms.post.slug` та інші
- **Два демо-плагіни:**
  - `ukr-to-lat` — автоматична транслітерація slug при збереженні запису
  - `seo-meta` — Open Graph і Twitter Card мета-теги для записів
- **`admin/ai_settings.php`** — окрема сторінка управління GROQ API Key (записує у `.env`)
- **`admin/run_migrations.php`** — утиліта для примусового повторного запуску міграцій після оновлення
- **Поле GROQ API Key в інсталяторі** (`install.php`, крок 3) — необов'язкове, записується у `.env` при встановленні
- **Пароль phpLiteAdmin на фінальному екрані** `install.php` — відображається одноразово після встановлення з попередженням «збережіть зараз»
- **phpLiteAdmin CSS вбудовано inline** — CSS читається через `file_get_contents()` і виводиться тегом `<style>`, оскільки `.htaccess` блокує прямий HTTP-доступ до файлів `SQLAdmin/`

### Виправлено
- `install.php` — схема БД повністю переписана відповідно до реальної структури (20 таблиць замість 16)
- `install.php` — виправлено помилку `table posts has no column named excerpt` у демо-контенті
- `data/migrations.php` — версія 9: додано 38 `ALTER TABLE ADD COLUMN` для всіх відсутніх колонок (`posts.visibility`, `posts.meta_title`, `post_revisions.title`, `post_revisions.saved_at`, `invitations.created_at`, `menu_items.visible`, `menu_items.type`, тощо)
- `admin/menu_editor.php` — запит `SELECT meta_title FROM posts` замінено на `COALESCE(meta_title, title)` щоб не падати на старих БД
- `admin/SQLAdmin/phpadmin.php` — виправлено відображення стилів: `file_exists()` тепер перевіряє відносно `__DIR__`, CSS виводиться inline

### Змінено
- `data/DbDriver.php` — повністю переписано: лише SQLite-утиліти (`now()`, `upsert()`, `insertIgnore()`, `tableExists()`, `getColumns()`, `walCheckpoint()`), вся MySQL-логіка видалена
- `config.php` — `fly_db()` є чистим SQLite singleton; додано `PRAGMA foreign_keys = ON`
- `install.php` — крок 2 «База даних» спрощено до інформаційного екрана (SQLite, нічого налаштовувати)
- `install.php` — базове меню після встановлення відповідає реальній структурі `menu_items`

### Видалено
- `admin/db_migrate.php` — майстер міграції SQLite → MySQL
- `data/mysql_schema.sql` — MySQL DDL схема
- MySQL badge, рядок PDO MySQL у вимогах, весь `.env` MySQL-блок — видалено з документації та інсталятора

## [2.7.0-AI] — 2026-03-15

### Додано
- **Система технічної підтримки** (`admin/support.php`) — тікет-система для спілкування адміністраторів з розробником:
  - Категорії звернень: баг, нова функція, безпека, хостинг, оновлення
  - Пріоритети: низький / звичайний / високий / критичний
  - Месенджер-інтерфейс з бульбашками повідомлень
  - Автоматичне надсилання email розробнику при кожному повідомленні
- **Відповіді розробника в CMS** (`admin/support_reply.php`) — публічна сторінка без логіну:
  - Розробник отримує одноразове посилання в листі
  - Відповідь зберігається в БД і одразу видна в CMS
  - Авто-оновлення переписки кожні 15 секунд (polling)
  - Нотифікація адміністратора листом про відповідь розробника
- **Налаштування SMTP через UI** (`admin/smtp_settings.php`):
  - Збереження SMTP параметрів прямо у `.env` без ручного редагування
  - Швидкий вибір провайдера (Gmail, Outlook, UKR.net, Meta.ua, SendGrid)
  - Підказки для кожного провайдера (App Password, увімкнення SMTP)
  - Тестова відправка листа
- **GitHub налаштування у smtp_settings.php** — секція для `GITHUB_OWNER`, `GITHUB_REPO`, `GITHUB_TOKEN`
- **Система оновлень з GitHub** (`admin/updater.php`):
  - Автоматична перевірка нових релізів при відкритті сторінки
  - Завантаження і розпакування ZIP архіву
  - Захист файлів: `.env`, `data/`, `uploads/`, `config.php` не перезаписуються
  - Автобекап БД перед кожним оновленням
  - Діагностика `ZipArchive`: показує шлях до `php.ini`, кнопка автоувімкнення
  - Ручне оновлення через URL архіву
- **Система запрошень з email** (`admin/invite.php`):
  - Email одержувача тепер обов'язковий
  - Автоматична відправка листа із запрошенням при наявності SMTP
  - Кнопка "↩ Ще раз" для повторної відправки
  - Бейдж "✉ надіслано" / "не надіслано" в таблиці
- **Єдина SMTP функція** (`admin/smtp_helper.php`):
  - Підтримка SSL (порт 465) через `ssl://` в fsockopen — виправлено для UKR.net, Meta.ua
  - Підтримка TLS (порт 587) через STARTTLS
  - Всі файли (support, invite, add_user, smtp_settings) використовують одну функцію
- **SMTP статус-бар** у `support.php`, `add_user.php`, `invite.php`

### Виправлено
- `config.php`: замінено `str_starts_with()` і `str_contains()` на сумісні з PHP 7.x варіанти
- `support.php`: виправлено зламану кнопку "Відповісти в CMS" у листі
- `support.php`: виправлено redirect після відправки повідомлення
- `smtp_helper.php`: SSL з'єднання тепер використовує `ssl://host` — виправлено для порту 465
- Кнопки в email-листах з inline CSS — не залежать від `<style>` блоків які Gmail ігнорує

### Змінено
- `admin_template.php`: додано пункти меню "🛠️ Підтримка" і "🔄 Оновлення CMS"
