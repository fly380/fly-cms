## [2.7.0-AI] — 2026-03-15

### Додано
- **Система технічної підтримки** (`admin/support.php`) — тікет-система для спілкування адміністраторів з розробником:
  - Категорії звернень: баг, нова функція, безпека, хостинг, оновлення
  - Пріоритети: низький / звичайний / високий / критичний
  - Месенджер-інтерфейс з бульбашками повідомлень
  - Автоматичне надсилання email розробнику при кожному повідомленні
- **Відповіді розробника в CMS** (`admin/support_reply.php`) — публічна сторінка без логіну:
  - Розробник отримує одноразове посилання в листі
  - Відповідь зберігається в БД і одразу видна в CMS
  - Авто-оновлення переписки кожні 15 секунд (polling)
  - Нотифікація адміністратора листом про відповідь розробника
- **Налаштування SMTP через UI** (`admin/smtp_settings.php`):
  - Збереження SMTP параметрів прямо у `.env` без ручного редагування
  - Швидкий вибір провайдера (Gmail, Outlook, UKR.net, Meta.ua, SendGrid)
  - Підказки для кожного провайдера (App Password, увімкнення SMTP)
  - Тестова відправка листа
- **GitHub налаштування у smtp_settings.php** — секція для `GITHUB_OWNER`, `GITHUB_REPO`, `GITHUB_TOKEN`
- **Система оновлень з GitHub** (`admin/updater.php`):
  - Автоматична перевірка нових релізів при відкритті сторінки
  - Завантаження і розпакування ZIP архіву
  - Захист файлів: `.env`, `data/`, `uploads/`, `config.php` не перезаписуються
  - Автобекап БД перед кожним оновленням
  - Діагностика `ZipArchive`: показує шлях до `php.ini`, кнопка автоувімкнення
  - Ручне оновлення через URL архіву
- **Система запрошень з email** (`admin/invite.php`):
  - Email одержувача тепер обов'язковий
  - Автоматична відправка листа із запрошенням при наявності SMTP
  - Кнопка "↩ Ще раз" для повторної відправки
  - Бейдж "✉ надіслано" / "не надіслано" в таблиці
- **Єдина SMTP функція** (`admin/smtp_helper.php`):
  - Підтримка SSL (порт 465) через `ssl://` в fsockopen — виправлено для UKR.net, Meta.ua
  - Підтримка TLS (порт 587) через STARTTLS
  - Всі файли (support, invite, add_user, smtp_settings) використовують одну функцію
- **SMTP статус-бар** у `support.php`, `add_user.php`, `invite.php`:
  - Зелений якщо активно, жовтий з деталями помилки якщо ні
  - Підказка про Gmail App Password і UKR.net пароль додатків
  - Пряме посилання на `smtp_settings.php`

### Виправлено
- `config.php`: замінено `str_starts_with()` і `str_contains()` на сумісні з PHP 7.x варіанти (`$line[0]`, `strpos()`)
- `email_config.php`: дефолт `SMTP_ENABLED` змінено з `'true'` на `'false'`
- `support.php`: виправлено зламану кнопку "Відповісти в CMS" у листі (heredoc псував PHP змінні)
- `support.php`: виправлено redirect після відправки повідомлення (тепер завжди зберігає `?ticket=ID`)
- `smtp_helper.php`: SSL з'єднання тепер використовує `ssl://host` замість plain `fsockopen` — виправлено для порту 465
- Кнопки в email-листах тепер з inline CSS (`color:#ffffff`) — не залежать від `<style>` блоків які Gmail ігнорує

### Змінено
- `add_user.php`, `invite.php`, `support.php`, `support_reply.php`, `smtp_settings.php`: весь SMTP-код замінено викликами `fly_smtp_send()` з `smtp_helper.php`
- `admin_template.php`: додано пункти меню "🛠️ Підтримка" і "🔄 Оновлення CMS"

Інтегровано ШІ-помічника (🤖) для генерації контенту. Розширено редактор сторінок підтримкою штучного інтелекту.

## [1.9]

Додано роль "редактор" з обмеженим доступом.

## [1.8.9]

Додано drag-and-drop редагування пунктів меню в редакторі меню. Пункти меню зберігаються в SQLite з полем `visibility_role` (user/admin). Підтримка вкладених (каскадних) меню.

## [1.8.4]

Додано підтримку Google reCAPTCHA v3 для входу. Можливість вмикати/вимикати reCAPTCHA з налаштувань.

## [1.8.0]

Покращення безпеки сесій: `session.cookie_httponly`, `secure`, `samesite=Lax`. Генерація CSRF-токенів у формах. Обмеження кількості спроб входу.

## [1.7.0]

Перехід із JSON на SQLite для зберігання сторінок. Міграція існуючих сторінок у `database.sqlite`.

## [1.6.0]

Сторінки мають статус: опубліковано / чернетка. Додано параметр видимості сторінки за ролями.

## [1.5.0]

Налаштування сайту: додано редактор заголовку, мета-тегів, футера. Додано завантаження логотипу, favicon, фонового зображення.

## [1.4.0]

Можливість редагування та видалення сторінок. Підтримка чернеток і прихованих сторінок. Обмеження доступу лише для авторизованих.

## [1.2.0]

Розмежування прав: адміністратор і звичайний користувач. Додано логування дій користувачів: вхід, вихід, створення сторінки. Додано базове навігаційне меню.

## [1.0.0]

Початковий реліз fly-CMS. Проста авторизація через `login.php` із локальними обліковими записами. Збереження сторінок у форматі JSON. Створення HTML-сторінок із заголовком і вмістом.
